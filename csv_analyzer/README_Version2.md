# CSV 文件分析工具

一个高性能的Python工具，用于分析超大型CSV文件（10G+）。可以快速计算文件的总行数，并分析前三列（或更多列）的数值范围。

## ✨ 特性

- **流式处理**: 逐行读取，不一次性加载整个文件到内存
- **超大文件支持**: 可处理10G+的文件，内存占用<100MB
- **自动编码检测**: 自动识别文件编码（UTF-8, GBK, Latin-1等）
- **数值范围分析**: 计算前N列的最小值、最大值、数据范围
- **类型检测**: 自动检测列的数据类型（int、float、string）
- **进度显示**: 实时显示处理进度和统计信息
- **灵活配置**: 支持自定义分隔符、编码、分析列数
- **多种输出**: 支持控制台和JSON格式输出
- **完善的错误处理**: 详细的日志记录和异常处理

## 📋 系统要求

- Python 3.7+
- chardet（用于编码检测）

## 💡 使用方法

### 命令行使用

#### 基本用法

```bash
python csv_analyzer.py data.csv
```

#### 高级用法

```bash
# 指定分隔符
python csv_analyzer.py data.csv --delimiter ";"

# 指定编码
python csv_analyzer.py data.csv --encoding utf-8

# 分析前5列而不是前3列
python csv_analyzer.py data.csv --analyze-columns 5

# 大文件分析（调整进度显示频率）
python csv_analyzer.py huge_file.csv --chunk-size 500000

# 导出为JSON格式
python csv_analyzer.py data.csv --json > result.json
```

### Python API 使用

```python
from csv_analyzer import CSVAnalyzer

# 创建分析器实例
analyzer = CSVAnalyzer(
    file_path='data.csv',
    delimiter=',',
    chunk_size=100000,
    analyze_columns=3
)

# 执行分析
result = analyzer.analyze()

# 访问结果
print(f"总行数: {result['total_rows']:,}")
print(f"数据行数: {result['data_rows']:,}")
print(f"处理速度: {result['rows_per_second']:,.0f} 行/秒")

# 获取列统计信息
for col_stats in result['columns_stats_detailed'].values():
    print(f"\n列: {col_stats['column_name']}")
    print(f"  最小值: {col_stats['min_value']}")
    print(f"  最大值: {col_stats['max_value']}")
    print(f"  有效数据: {col_stats['valid_ratio']}")
```

## 🔧 参数说明

| 参数 | 说明 | 默认值 | 类型 |
|------|------|--------|------|
| `file_path` | CSV文件路径 | 必需 | str |
| `delimiter` | CSV分隔符 | `,` | str |
| `encoding` | 文件编码 | 自动检测 | str |
| `chunk_size` | 显示进度的行数间隔 | 100000 | int |
| `analyze_columns` | 分析的列数 | 3 | int |

## 🛠️ 内部工作原理

### 1. 编码检测
- 读取文件前100KB进行编码检测
- 支持UTF-8、GBK、Latin-1等多种编码

### 2. 流式读取
- 使用Python csv模块的迭代器
- 逐行处理，不缓存整个文件

### 3. 数值转换
- 尝试将字符串转换为整数或浮点数
- 跳过非数值数据，记录有效数据比率

### 4. 范围计算
- 使用最小/最大值追踪算法
- 避免重复类型转换，效率高

### 5. 类型检测
- 对前1000行有效数据进行采样
- 判断是整数还是浮点数

## ⚠️ 注意事项

1. **文件编码**: 工具自动检测编码。如果检测不正确，使用 `--encoding` 参数指定。

2. **分隔符**: 默认为逗号。如果使用其他分隔符（如分号、制表符），使用 `--delimiter` 参数。

3. **表头**: 工具假设第一行是表头。如果没有表头，结果可能不准确。

4. **大型文件**: 对于超大文件，可能需要几分钟到十几分钟。进度显示可帮助了解进度。

5. **内存占用**: 工具内存占用与文件大小无关，仅与数据类型和分析列数有关。

## 🐛 故障排除

### 问题1: 编码错误（乱码）

**症状**: 输出中出现乱码或编码错误

**解决**:
```bash
# 尝试指定正确的编码
python csv_analyzer.py data.csv --encoding gbk
# 常见编码: utf-8, gbk, latin-1, cp1252
```

### 问题2: 数值识别不正确

**症状**: 某列应该是数值但被识别为字符串

**解决**:
- 检查数据是否包含非数值字符（如空格、特殊符号）
- 查看日志文件了解具体情况

### 问题3: 处理速度慢

**症状**: 处理速度明显低于预期

**解决**:
- 检查磁盘I/O是否成为瓶颈
- 增加 `--chunk-size` 以减少进度显示开销
- 确保足够的可用内存
